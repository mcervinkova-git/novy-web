Automatizujte reporty: BigQuery + Looker Studio bez programování

18. dubna 2026 · Markéta Červinková

Každý týden stahovat data z Google Ads do Excelu, překlápět je do tabulky a posílat klientovi PDF — to byl můj život ještě před dvěma lety. Dnes mám pipeline, který data stahuje automaticky každý den, ukládá je do BigQuery a Looker Studio je zobrazuje v dashboardu, který si klient otevře sám, kdykoli chce. A celé nastavení zabralo méně než den.

Proč přestat exportovat ručně

Ruční reporty mají tři zásadní problémy:

  • Jsou zpožděné. Klient vidí data z minulého týdne, ne z dnešního rána.
  • Jsou chybové. Jeden špatný VLOOKUP a čísla nesedí.
  • Žerou čas. Hodiny týdně, které byste mohli věnovat skutečné optimalizaci.

Automatizovaný pipeline tyto problémy řeší najednou. A nejlepší část: Google poskytuje všechny potřebné nástroje zdarma nebo za minimální poplatek.

Krok 1: Google Ads Data Transfer do BigQuery

BigQuery Data Transfer Service umí přenášet data z Google Ads přímo do BigQuery tabulek — automaticky, každý den. Nastavení zvládnete přes Google Cloud Console bez jediného řádku kódu:

  1. V Google Cloud Console otevřete BigQuery a pak BigQuery Data Transfer Service.
  2. Vyberte zdroj „Google Ads" a propojte svůj účet.
  3. Zvolte dataset (nebo vytvořte nový) a nastavte frekvenci přenosu (doporučuji denně).
  4. Vyberte, která data chcete přenášet — pro PPC reporting zpravidla stačí Campaign, AdGroup, Ad a Keyword tabulky.

Přenos začne automaticky a data budou dostupná v BigQuery do 24 hodin.

Tip: BigQuery Data Transfer je zdarma co do přenosu samotného — platíte pouze za dotazy a uložení dat v BigQuery. Pro běžný PPC účet jsou náklady v řádu jednotek korun měsíčně.

Krok 2: Základní SQL dotaz pro dashboard

Jakmile máte data v BigQuery, potřebujete je "přeložit" do formy vhodné pro Looker Studio. Tady přichází SQL — ale nebojte se, základní dotazy jsou jednoduché. Příklad dotazu pro přehled výkonu kampaní: 

SELECT
  campaign_name,
  SUM(impressions)   AS impressions,
  SUM(clicks)        AS clicks,
  SUM(cost) / 1e6   AS cost_czk,
  SUM(conversions)   AS conversions,
  SUM(cost) / 1e6 / NULLIF(SUM(conversions), 0) AS cpa
FROM
  `vas-projekt.ads_dataset.p_campaign_*`
WHERE
  _DATA_DATE >= DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY)
GROUP BY
  campaign_name
ORDER BY
  cost_czk DESC;

Výsledek tohoto dotazu ukáže výkon každé kampaně za posledních 30 dní — včetně CPA spočítaného automaticky. Dotaz uložte jako View v BigQuery a Looker Studio ho pak bude volat přímo.

Krok 3: Dashboard v Looker Studiu

Looker Studio je zdarma a propojení s BigQuery je nativní — stačí kliknout na „Přidat data" a vybrat váš projekt. Pak:

  • Přidejte grafy (výkonnostní trendová čára, tabulka kampaní, scorecardy pro klíčové metriky).
  • Nastavte ovládací prvky data — filtr na rozsah dat, který si klient může měnit sám.
  • Sdílejte dashboard odkazem — klient ho otevře v prohlížeči a vidí vždy aktuální data.

Výsledek

Po tomto nastavení máte živý dashboard, který se aktualizuje každý den bez vaší účasti. Klient má přehled kdykoli chce, vy šetříte hodiny každý týden a reporty přestávají být zdrojem chyb. Win-win.

Pokud vás toto téma zajímá nebo chcete podobný pipeline nastavit pro svůj účet, napište mi — ráda pomůžu.
